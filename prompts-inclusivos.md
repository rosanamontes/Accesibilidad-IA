## Anatomía de un Prompt inclusivo

Para crear prompts que realmente favorezcan la inclusión de tus alumnos con Necesidades Específicas de Apoyo Educativo (NEAE), es fundamental entender que la IA no es accesible por defecto y puede mejorar gracias a tu rol como docente.
Saber dar las instrucciones precisas al modelo de IA generativa lo es todo. La calidad de las respuestas proviene de la calidad del *promt*.

1. Define explícitamente el nivel de lectura y complejidad
Por defecto, modelos como ChatGPT suelen generar respuestas con un nivel de legibilidad de estudiante universitario.
Paso: Incluye siempre en el prompt el nivel educativo específico. Por ejemplo: "Explica las leyes de Newton para un alumno de 6.º de primaria utilizando lenguaje sencillo y oraciones cortas".

2. Solicita estructuras semánticas y ayudas técnicas (Evita el "Vibe Coding")
La IA a menudo prioriza la estética sobre la estructura técnica funcional.
Paso: Si pides a la IA que genere código o contenido web para tus alumnos, solicita explícitamente etiquetas semánticas (como <nav>, <h1>), navegación por teclado y compatibilidad con lectores de pantalla. Esto evita que el contenido sea visualmente atractivo pero inaccesible.

3. Desglosa tareas complejas en pasos manejables
La carga cognitiva de redactar o entender instrucciones largas puede frustrar a alumnos con dificultades de aprendizaje o discapacidades cognitivas.
Paso: Utiliza el "andamiaje" dividiendo el prompt en partes. En lugar de una tarea única, pide a la IA que guíe al alumno paso a paso: "Primero, dame un esquema. Cuando yo diga 'siguiente', explica el primer punto".

4. Utiliza feedback estructurado para corregir errores
Cuando la IA no responda de forma inclusiva, no uses reintentos genéricos; usa una estructura de tres puntos para refinar el resultado.
Paso: Indica a la IA:
Acción realizada: "Te pedí una explicación visual".
Resultado esperado: "Esperaba descripciones de texto alternativo para un alumno ciego".
Resultado real: "Solo me diste el código de la imagen".

5. Incorpora el razonamiento visual si es posible
Si utilizas herramientas con capacidad de visión (como GPT-4o), el apoyo visual mejora drásticamente la accesibilidad.
Paso: Proporciona capturas de pantalla de los materiales que usas en clase y pide a la IA que analice si el contraste de color es suficiente o que genere descripciones detalladas (Alt Text) de las imágenes para tus alumnos con discapacidad visual.

6. Pide a la IA que refine tus propios prompts para los alumnos
La ingeniería de prompts puede ser una barrera adicional para el estudiante.
Paso: Tú puedes redactar un prompt complejo y pedirle a la IA: "Simplifica este prompt para que un alumno con TDAH pueda entender las instrucciones sin perderse en los detalles".

7. Supervisión humana constante (Human-in-the-loop)
Las fuentes enfatizan que el criterio del experto (el docente) es irreemplazable para detectar "alucinaciones" o tonos inapropiados que la IA no percibe como errores técnicos.
Paso: Siempre valida la "sabiduría" y el contexto humano de la respuesta antes de entregarla al alumno, asegurándote de que no contenga sesgos capacitistas que traten la discapacidad como una desviación.
